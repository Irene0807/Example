# Skill: Map News to Game Theory

## 觸發時機
當使用者說「分析這則新聞」、「這則新聞對應什麼賽局」或使用 `/analyze-headline` 指令時。

## 概念對應表

| 關鍵詞 | 賽局類型 | 核心概念 |
|--------|----------|----------|
| 談判、協議、條約 | 討價還價賽局 | 納許談判解、外部選項 |
| 制裁、貿易戰、關稅 | 囚徒困境 | 相互報復、合作困境 |
| 選舉、競選、政黨 | 協調賽局 | 焦點均衡、選民預期 |
| 聯盟、結盟、合作 | 合作賽局 | 核心解、沙普利值 |
| 軍備、武力、威懾 | 膽小鬼賽局 | 可信威脅、承諾機制 |
| 競標、拍賣、標案 | 拍賣理論 | 贏家詛咒、出價策略 |
| 價格競爭、市場占有 | 古諾 / 伯川德競爭 | 反應函數、納許均衡 |
| 氣候、公共財、共有財 | 公共財賽局 | 搭便車問題、集體行動 |

## 執行步驟

### Step 1：識別參與者
- 誰是決策者？（國家、企業、個人、政黨）
- 各方的核心利益是什麼？

### Step 2：判斷賽局類型
依照上方對應表，比對標題關鍵詞，選出最符合的賽局類型。
若同時匹配多個，選擇最能解釋「衝突核心」的那個。

### Step 3：建立報酬結構
```
參與者 A：[合作 / 競爭 / 退讓]
參與者 B：[合作 / 競爭 / 退讓]

預期報酬矩陣（定性）：
              B合作        B競爭
A合作    (中，中)      (低，高)
A競爭    (高，低)      (差，差)
```

### Step 4：預測均衡結果
- 是否有優勢策略？
- 納許均衡在哪裡？
- 均衡是否為帕累托最優？

### Step 5：輸出格式
```markdown
**新聞：** [標題]

**賽局類型：** [類型名稱]

**參與者：** A = [身份]；B = [身份]

**核心矛盾：** [一句話描述個人理性 vs 集體理性的張力]

**均衡預測：** [預測結果及理由]

**現實驗證：** [歷史案例或可觀察的指標]
```

## 注意事項
- 語言：全程繁體中文
- 術語：遵守 `rules/zh-tw.md` 的標準化術語
- 準確性：遵守 `rules/accuracy.md`，不得將均衡與最優混用
